# 06 – Control Mapping

---

## Overview

This section connects identified risks with selected security controls and relevant compliance frameworks.

It provides a structured mapping between:

- risks identified in the risk assessment
- controls defined in the ISMS
- external standards and frameworks

---

## Purpose

The objective of control mapping is to ensure:

- traceability between risks and controls
- alignment with recognized standards
- transparency for audits and compliance checks

---

## Approach

Controls are mapped against:

- ISO 27001 Annex A controls
- selected regulatory or industry frameworks (e.g., NIST, NIS2)

Each mapping ensures that:

- risks are adequately covered by controls
- controls can be justified in an audit context
- overlaps and gaps are visible

---

## Structure

The mapping includes:

- risk reference
- control reference (ISO 27001)
- mapped framework (e.g., NIST, NIS2)
- implementation status

---

## Outcome

The result is a transparent and structured mapping that:

- links risk management to compliance requirements
- supports audit readiness
- enables gap analysis and continuous improvement

---

## Note

The detailed mapping matrix is maintained in a separate file and reflects a simplified but realistic implementation scenario.
