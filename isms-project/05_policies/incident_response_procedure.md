## Incident Response Procedure

### Objective

This procedure defines the operational steps for handling security incidents.

---

### Incident Classification

| Severity | Description | Example |
|----------|------------|--------|
| Low      | Minor issue | Single endpoint malware |
| Medium   | Limited impact | Partial system outage |
| High     | Major disruption | Ransomware, data breach |

---

### Response Timeline

| Phase        | Target Time |
|--------------|------------|
| Detection    | Immediate  |
| Classification | < 1 hour |
| Containment  | < 4 hours |
| Recovery     | < 24 hours |

---

### Step-by-Step Process

#### 1. Detection
- Monitor alerts (SIEM, logs, user reports)
- Validate incident

#### 2. Initial Response
- Inform IT team
- Escalate to management if required

#### 3. Containment
- Isolate affected systems
- Disable compromised accounts

#### 4. Investigation
- Identify root cause
- Assess impact

#### 5. Recovery
- Restore systems from backup
- Verify system integrity

#### 6. Communication
- Internal notification
- External reporting if required (GDPR, NIS2)

#### 7. Lessons Learned
- Document findings
- Update controls and processes
