# ISMS Mini Implementation – Automotive Supplier

---

## Overview

This project represents the design of a structured, ISO 27001-aligned Information Security Management System (ISMS) for a mid-sized automotive supplier in Germany.

The ISMS is designed as a practical governance framework that integrates risk management, control definition, and operational processes into a coherent and business-aligned system.

---

## Objective

To design an ISMS that enables:

- identification and prioritization of business-relevant risks  
- alignment between security controls and operational requirements  
- clear ownership and governance structures  
- implementation of effective and feasible security measures  

---

## Scenario Context

The simulated organization operates in a production-driven environment with:

- high dependency on system availability  
- integration with external suppliers  
- handling of sensitive operational and customer data  
- increasing reliance on cloud-based collaboration tools  

---

## ISMS Structure

The ISMS is structured around:

- Scope definition and system boundaries  
- Asset identification and classification  
- Risk assessment and prioritization  
- Control selection based on risk exposure  
- Policy framework for governance  
- Statement of Applicability (SoA)
